📤 GitHub에 파일 업로드하는 방법

Git 설치 없이 브라우저만으로 GitHub에 파일을 올리는 3가지 방법을 단계별로 설명합니다.

가장 쉬운 방법. 파일을 그냥 브라우저 창으로 끌어다 놓으면 끝입니다.
여러 파일을 한꺼번에 올릴 수 있습니다.
1

저장소 열기

github.com에서 업로드할 폴더로 이동
  • ①

    github.com 접속 → 로그인

  • ②

    저장소 rpaakdi1-spec / cargo-driver-app 클릭

  • ③

    파일을 올릴 폴더로 이동
    예) android/app/src/main/java/com/cargo/driver/ 폴더로 들어가기

github.com/rpaakdi1-spec/cargo-driver-app
rpaakdi1-spec / cargo-driver-app Public
android / app / src / main / java / com / cargo / driver
..상위 폴더
GpsService.java2 days ago
MainActivity.java2 days ago
WebAppInterface.java2 days ago
2

Add file → Upload files 클릭

폴더 화면 오른쪽 상단 버튼
github.com/rpaakdi1-spec/cargo-driver-app/tree/main/android
android /
Create new file
Upload files ← 이거!
3

파일을 드래그해서 놓기

점선 박스 안으로 파일을 끌어다 놓거나, "choose your files" 클릭
github.com/rpaakdi1-spec/cargo-driver-app/upload/main/android
Drag files here to add them to your repository
or choose your files
GpsService.java — 업로드 완료 (7.2 KB)
주의: 이미 같은 이름의 파일이 있으면 덮어씁니다.
업로드 전에 기존 내용을 백업하고 싶다면 먼저 파일 내용을 복사해두세요.
4

커밋 메시지 입력 후 저장

페이지 아래 "Commit changes" 영역
github.com/…/upload

Commit changes

  • ①

    첫 번째 칸: 간단한 설명 입력 (예: GpsService GPS 주기 10초로 변경)

  • ②

    두 번째 칸: 상세 설명 (선택, 비워도 됨)

  • ③

    "Commit changes" 초록 버튼 클릭

  • ④

    자동으로 Actions 탭에서 APK 빌드 시작 (약 5~7분)

파일을 새로 만들거나, 내용 전체를 복사해서 붙여넣을 때 사용합니다.
파일이 없는 새 경로에 파일을 만들 때도 이 방법을 씁니다.
1

Add file → Create new file

저장소 아무 폴더에서나 가능
  • ①

    저장소 메인 또는 원하는 폴더 이동

  • ②

    Add file 버튼 → Create new file 클릭

2

파일 이름에 경로 포함해서 입력

슬래시(/)를 입력하면 자동으로 폴더가 생성됩니다
github.com/rpaakdi1-spec/cargo-driver-app/new/main
cargo-driver-app / android/app/src/main/java/com/cargo/driver/GpsService.java

💡 슬래시(/)를 입력하면 폴더가 자동 생성됩니다.
예: android/ 입력 → android 폴더 생성

파일 이름 칸에 경로/파일명 전체를 입력하면 됩니다.
예: android/app/src/main/java/com/cargo/driver/GpsService.java
3

내용 붙여넣기

아래 파일별 전체 코드를 복사해서 에디터에 붙여넣기
github.com/…/new/main
1
2 package com.cargo.driver;
3
4 import android.app.Service;
5 ... (코드 계속)
Ctrl+A로 전체 선택 후 Ctrl+V로 붙여넣기
기존 파일을 수정할 때는 이 방법보다 파일 직접 열어서 연필 아이콘 클릭이 더 편합니다.
이 방법은 완전히 새 파일을 만들 때 주로 사용하세요.
android 폴더 전체를 처음 업로드하거나, 여러 폴더를 한 번에 올릴 때 사용합니다.
1

저장소 루트(최상단)에서 시작

폴더 구조 그대로 유지됩니다
  • ①

    저장소 첫 페이지(루트) 이동
    github.com/rpaakdi1-spec/cargo-driver-app

  • ②

    Add file → Upload files 클릭

2

폴더를 드래그해서 놓기

파일 탐색기에서 폴더 자체를 끌어다 놓으면 됩니다
github.com/rpaakdi1-spec/cargo-driver-app/upload/main
폴더를 여기로 드래그
android 폴더 전체를 끌어다 놓기
android/app/build.gradle
android/app/src/main/AndroidManifest.xml
android/app/src/main/java/com/cargo/driver/GpsService.java
+ 8개 파일 더...
폴더를 드래그하면 하위 파일 모두 자동으로 올라갑니다.
폴더 구조(경로)도 그대로 유지됩니다.
크롬/엣지 브라우저 권장. 일부 브라우저는 폴더 드래그가 안 될 수 있습니다.
안 되면 방법 1(파일 하나씩)이나 방법 2를 사용하세요.

APK 기사님 기기에 설치

카카오톡으로 전송 또는 링크 공유
  • APK 파일 다운로드 후 카카오톡으로 기사님께 전송
  • 기사님 폰에서 파일 열기 → "알 수 없는 출처 허용" → 설치
  • 기존 앱이 설치되어 있으면 삭제 없이 덮어쓰기 설치 가능 (단, versionCode가 더 커야 함)
  • 설치 후 앱 실행 → GPS 권한 "항상 허용" 설정
deploy-guide.html의 APK 다운로드 탭을 설정하면 기사님이 직접 사이트에서 최신 APK를 다운로드할 수도 있습니다.

업로드 시 자주 생기는 문제

  • Actions 탭이 없다
    Settings → Actions → General → "Allow all actions" 선택 후 Save
  • 빌드가 ❌ 빨간색으로 실패
    실패한 워크플로 클릭 → 오류 메시지 확인. 보통 Java 문법 오류거나 versionCode 중복입니다.
  • Artifacts(APK)가 없다
    build.yml 파일에 업로드 설정이 없는 경우입니다. deploy-guide.html의 GitHub Actions 빌드 탭을 참고하세요.
  • 폴더 드래그가 안 된다
    크롬 브라우저를 사용하세요. 또는 방법 1(파일 하나씩)을 이용하세요.
✅ 완료!
