# 화물운송 관리 시스템 — 사용방법 완전 가이드

> **버전**: v20250321T | **최종 수정**: 2025-03-21

---

## 📌 시스템 개요

화물 운송업체를 위한 3자 연동 웹 시스템입니다.

| 역할 | 담당 | 접속 URL |
|------|------|----------|
| 🛡️ **관리자** | 고객사 룸 생성·전체 현황 관리 | `admin.html` |
| 🏢 **고객사** | 배송 등록·서류 확인·GPS 추적·알림 수신 | `room.html?id=ROOM_ID` |
| 🚛 **기사** | GPS 공유·서류 사진 업로드·추가사진 | `driver.html` |
| 🖼️ **갤러리** | 전체 서류 사진 조회 | `gallery.html` |
| 📱 **QR** | 역할별 접속 QR 생성 | `qr.html` |

## 1️⃣ 관리자 사용법 (`admin.html`)

### 로그인
- 비밀번호: `rhkdtls1`
- 세션: 4시간 유지
- ⚠️ **5회 연속 실패 시 60초 잠금** (이후 실패마다 2배씩 최대 30분)

### 고객사 룸 관리
1. **고객사 관리** 탭 → **룸 생성** 버튼
2. 고객사명, 비밀번호(고객사가 사용할 비밀번호), 연락처, 설명 입력
3. **저장** 클릭 → 룸 ID 자동 생성
4. 생성된 룸의 **눈 아이콘** 클릭 → 고객사 전용 URL 확인
5. URL을 고객사 담당자에게 카카오톡으로 전달

### 전체 배송 현황
- **전체 배송 현황** 탭 → 모든 고객사 배송 일괄 조회
- 상태 변경(긴급 수정 시): 편집 아이콘 클릭 → 상태 직접 입력
- 배송 삭제: 휴지통 아이콘 클릭

### 🔔 실시간 알림
- 헤더 우상단 **🔔 벨 아이콘** → 알림 패널 열기
- 새 알림 발생 시 뱃지 숫자 표시 + 토스트 팝업
- 수신 알림 종류:

| 아이콘 | 이벤트 | 색상 |
|--------|--------|------|
| 🚛 | 기사 앱 접속 | 파랑 |
| 📦 | 상차 완료 | 주황 |
| 📍 | 하차지 도착·하차대기 | 황금 |
| ✅ | 전체 하차 완료 | 초록 |
| ⚠️ | 저속 장시간 지속 (15km/h 미만 40분) | 빨강 |
| ⏰ | 상차 후 4시간 경과·미하차 | 보라 |

### 📷 이미지 서버 설정 (중요! 500 에러 방지)
> 기사님 사진이 DB에 쌓이면 500 에러 발생 → UVIS 서버 연동 필수

1. **설정** 탭 이동
2. **UVIS 아이디 / 비밀번호** 입력 (rhkdtls.cloud 계정)
3. **계정 저장** → **연결 테스트** → ✅ 성공 확인
4. **기사용 링크 생성** → 링크 복사 → 기사님께 카카오톡 전송
5. 기사님이 해당 링크로 접속 시 자동 설정 완료

### 사진 보관함
- 헤더 우상단 **📷 사진 보관함** 클릭 → `gallery.html`
- 날짜별·거래처별·전체 그리드 보기
- 필터: 거래처, 날짜 범위, 기사명, 서류 유형

## 2️⃣ 고객사 사용법 (`room.html?id=ROOM_ID`)

### 로그인
- 관리자에게 받은 URL로 접속
- 비밀번호 입력 (관리자가 설정한 비밀번호)
- 세션: 8시간 유지

### 배송 등록
1. **+ 새 배송 등록** 버튼 클릭
2. 아래 정보 입력:

| 항목 | 설명 |
|------|------|
| **기사명** | 배송 담당 기사 이름 (정확히 입력!) |
| **차량번호** | 예) 12가 3456 |
| **기사 핸드폰** | 숫자 입력 시 자동 하이픈 (예: 010-1234-5678) |
| **화물 타입** | ❄️ 냉동/냉장 · 🧊 냉동 · 🌡️ 냉장 · 📦 상온 선택 |
| **출발지 / 목적지** | 주소 입력 |
| **기사 PIN** | 기사님이 로그인할 4~8자리 숫자 |
| **메모** | 특이사항 (선택) |

3. **배송 등록** 클릭

> ⚠️ **기사 PIN은 기사님과 미리 협의 필수!** 기사님이 driver.html에서 이 PIN으로 로그인합니다.

> 🌡️ **냉동/냉장 화물**은 기사 앱에서 온도기록지 업로드 시 **AB온도 확인 안내**가 자동 표시됩니다.

### 배송 현황 확인
- **배송 목록** 탭: 전체 배송 카드 보기
- 상태 필터 버튼: 전체 / 대기 / 상차완료 / 운송중 / 배송완료
- 검색창: 기사명·차량번호·화물 검색
- 카드에 기사 전화번호 표시 (등록 시)

### 배송 상세 확인
배송 카드 클릭 → 상세 모달 오픈:
- **서류** 탭: 상차 거래명세표·온도기록지 + 추가사진 + 하차지별 서류
- **지도** 탭: 기사님 실시간 GPS 위치 (30초 자동 갱신)
  - 📡 **현재 위치 요청** 버튼: 기사님 앱에 즉시 위치 전송 요청
  - 📸 **촬영 요청** 버튼: 미촬영 서류 항목에서 기사님께 촬영 요청 발송

### 🔔 실시간 알림
- 헤더 우상단 **🔔 벨 아이콘** 클릭 → 알림 패널 (내 룸 배송 알림만 표시)
- 10초 간격 자동 폴링
- 알림 유형: 기사 접속 / 상차완료 / 도착대기 / 하차완료 / 저속경고 / 4시간경과

## 3️⃣ 기사 사용법 (`driver.html`)

### 최초 설정 (1회만)
> 관리자가 보내준 **기사용 링크**로 최초 접속하면 이미지 업로드 설정이 자동 완료됩니다.

### 로그인
1. **기사명** 입력 (고객사에서 등록한 이름과 **동일하게**)
2. **PIN** 입력 (고객사 담당자에게 받은 숫자 4~8자리)
3. **로그인** 클릭

> ⚠️ 이름이 조금이라도 다르면 로그인 불가
> ⚠️ **5회 연속 실패 시 30초 잠금** (이후 실패마다 2배씩 최대 10분)

### 업무 시작
1. 로그인 성공 → 담당 배송건 선택 (화물·출발지·목적지·상태 표시)
2. **업무 시작** 버튼 클릭
3. 업무 화면으로 이동 (관리자·고객사에 **기사 접속 알림** 자동 전송)

### GPS 공유
1. **GPS 시작** 버튼 클릭
2. 위치 권한 허용 (반드시 **"항상 허용"** 선택)
3. GPS 상태가 🟢 켜짐으로 변경
4. 30초마다 자동으로 서버에 위치 전송 (현재 속도 포함)
5. 화주가 위치 요청 시 즉시 전송
6. 앱 화면 이탈 후 복귀 시 GPS 자동 재연결

> 📱 **화면을 끄거나 네비게이션 앱 사용 시**: APK를 사용하면 백그라운드에서도 GPS 유지됩니다.

### 상차 완료 처리
1. 상차 후 **거래명세표** 촬영 → 자동 업로드
2. **온도기록지** 촬영 → 자동 업로드
   - ❄️ 냉동/냉장 화물의 경우 **"AB온도 확인 필수"** 안내 자동 표시
3. (선택) **[+ 사진 추가]** 버튼으로 물품사진·현장사진 추가 촬영
   - 한 번에 여러 장 선택 가능
   - 썸네일 우상단 ❌ 버튼으로 삭제 가능
4. **상차 완료 처리** 버튼 클릭 → 상태 '상차완료' + **관리자·고객사 알림 전송**
5. 상차 취소가 필요한 경우 → **상차 취소** 버튼 클릭 → 확인 후 대기 상태로 복귀

### 하차 완료 처리
1. 하차지 도착 시 **[하차지 도착 · 하차 대기]** 버튼 클릭
   - 도착 시각이 기록되고 **관리자·고객사에 도착 알림 전송**
   - 상차 4시간 경과 알림 자동 중단
2. 하차 준비 완료 후 **거래명세표** 촬영
3. **온도기록지** 촬영
4. (선택) **[+ 사진 추가]** 버튼으로 현장사진 추가 촬영
5. **하차 완료 처리** 버튼 클릭 → **관리자·고객사 하차완료 알림 전송**

### 경유(다중 하차) 처리
1. **경유 하차지 추가** 버튼 클릭 → 새 하차지 세트 추가
2. 하차지 이름 수정 가능 (예: "2차 하차지 — ○○마트")
3. 각 하차지마다: **도착대기 버튼** → 서류 촬영 → **하차완료 버튼**
4. 모든 하차지 완료 시 → 전체 상태 **배송완료** 자동 전환 + GPS 자동 종료

### 화주 촬영 요청 수신
- 화주가 서류 촬영을 요청하면 → 앱 상단에 **빨간 배너 팝업** 표시
- "○○ 사진을 촬영해 주세요!" 문구 + 토스트 알림
- 30초 후 자동으로 배너 사라짐

### 세션 복원 (앱/브라우저 재시작 후)
- 12시간 이내 재접속 시 이전 업무 자동 복원
- GPS·위치요청폴링·사진요청폴링·저속감지 타이머 모두 자동 재시작
- 토스트 메시지: "📍 이전 업무 복원! GPS 자동 재시작"

## 4️⃣ 자동 알림 시스템

### 알림이 발송되는 시점

| 이벤트 | 기사 앱 동작 | 수신자 |
|--------|-------------|--------|
| 기사 앱 접속 | 업무 시작 버튼 클릭 | 관리자 + 해당 고객사 |
| 상차 완료 | 상차완료 버튼 클릭 | 관리자 + 해당 고객사 |
| 하차지 도착 | 도착·하차대기 버튼 클릭 | 관리자 + 해당 고객사 |
| 전체 하차 완료 | 마지막 하차지 완료 처리 | 관리자 + 해당 고객사 |
| ⚠️ 저속 장시간 | GPS 속도 15km/h 미만 40분 지속 후 → **속도 회복 전까지 5분마다 반복** | 관리자 + 해당 고객사 |
| ⏰ 상차 4시간 경과 | 상차 후 4시간 경과 + 도착대기 미처리 → **도착대기 버튼 누를 때까지 5분마다 반복** | 관리자 + 해당 고객사 |

> 💡 **저속 알림**: 속도가 15km/h 이상으로 회복되면 알림 자동 중단  
> 💡 **4시간 경과 알림**: 기사님이 도착·하차대기 버튼을 누르면 즉시 중단

## 6️⃣ 자주 묻는 질문 (FAQ)

### Q. 기사님이 로그인이 안 돼요
**A.** 기사명이 고객사에서 등록한 이름과 **정확히 동일**해야 합니다.
- 공백, 띄어쓰기도 구분합니다
- 예) 고객사 등록: `홍길동` → 기사 입력: `홍길동` (O), `홍 길동` (X)
- PIN도 고객사에서 설정한 번호와 동일해야 합니다
- 5회 실패 시 잠금 — 잠금 해제까지 기다리세요

### Q. 배송건이 목록에 안 보여요
**A.** 아래를 확인하세요:
- 기사명이 정확한지 확인
- 배송 상태가 `배송완료`면 목록에 안 보임 → 상태 필터에서 '완료' 선택
- 고객사 담당자에게 PIN 재확인

### Q. GPS가 자꾸 꺼져요
**A.**
- 위치 권한을 **"항상 허용"**으로 설정 (설정 → 앱 → 화물운송기사 → 위치)
- 배터리 최적화에서 해당 앱 **"제한 없음"**으로 설정
- 네비게이션 사용 시 앱 백그라운드 유지 (홈 버튼으로 최소화)
- Android APK 설치 시 백그라운드 GPS가 더 안정적

### Q. 저속 알림이 계속 와요
**A.** 기사님이 장시간 저속(15km/h 미만) 상태임을 의미합니다.
- 정체·대기 중이라면 정상 알림입니다
- 속도가 회복되면 자동으로 알림이 중단됩니다
- 하차지 도착 시 **도착·하차대기 버튼**을 눌러야 4시간 알림이 중단됩니다

### Q. 사진 업로드가 안 돼요
**A.**
- 관리자에게 **기사용 링크**를 다시 받아서 접속하세요
- 인터넷 연결 상태 확인
- 사진 크기는 자동으로 최적화됩니다 (20MB 이하면 정상)

### Q. 500 에러가 발생해요
**A.** 이미지가 DB에 직접 저장되어 서버 메모리 초과 상태입니다.
- 관리자 → **설정** 탭 → UVIS 계정 등록 → 연결 테스트
- 기사용 링크 생성 후 기사님께 전달
- 설정 완료 후 신규 사진은 외부 서버에 저장됩니다
